题目描述

原题连接

给你一个 m x n 的二元矩阵 matrix ，且所有值被初始化为 0 。请你设计一个算法，随机选取一个满足 matrix[i][j] == 0 的下标 (i, j) ，并将它的值变为 1 。所有满足 matrix[i][j] == 0 的下标 (i, j) 被选取的概率应当均等。

尽量最少调用内置的随机函数，并且优化时间和空间复杂度。

实现 Solution 类：

Solution(int m, int n) 使用二元矩阵的大小 m 和 n 初始化该对象int[] flip() 返回一个满足 matrix[i][j] == 0 的随机下标 [i, j] ，并将其对应格子中的值变为 1
void reset() 将矩阵中所有的值重置为 0

提示：

  • 1 <= m, n <= 10000
  • 每次调用flip 时，矩阵中至少存在一个值为 0 的格子。
  • 最多调用 1000 次 flip 和 reset 方法。

解题思路

1.暴力解决

观察题干，发现只需要对于一个二维矩阵的节点进行一个0与1的变换，题干思路非常清晰 开始嘎嘎写，然后发现报内存 ，但仔细看一眼数据,然后计算空间复杂度是 ，暴力算法必定会超过算法的空间限制。

优化1：哈希表

由上面的分析可知，将矩阵状态存储下来必然会爆空间，所以存储矩阵是不可能的。通过观察题干发现flip的操作次数最大只有 次，所以比起存储整个矩阵的状态，存储被flip操作过的数据明显更加划算。

由于flip操作的数据是随机分布的，所以只使用数据存储flip操作过的数据的话，查询的时间复杂度极大，为 可以想象一下一个不连续，不单调的数组，每次查都要遍历一遍，可以使用平衡树的方法去优化数据的查询但一道中等单独的题使用平衡树😂

或者，我们也可以是同哈希表去存储数据，它在时间和空间上都有较好的性能。

优化2：随机化

通过哈希表的优化，通过题目测试必然不在话下，但我们思考一下，比如是 的矩阵，当你反转了个左右，到后面你要找到一个没有反转的就需要多次的 rand 了,所以在生成随机数的角度上还可以优化。
首先，我们生成随机数所在的范围区间是连续的，但由于我们flip后的数据不应该在被选中，所以我们实际的区间是不连续且不可控的，这也导致我们可能需要生成多次随机数。
如下图，若我们生成了一次5的话，若在生成一次5的话，我们就需要再次生成，

所以我们可以将抽过的数据与末尾的数据交换去提高随机数的生成效率。如下图，我们可以将数据理解为一个指针的形式，由地址x指向数据y，在最初x等于y。而我们rand抽去的数据是地址。所以若我们随机生成了5，那我们就将5指向的数据与末尾指向的数据交换，将rand的范围减去末尾，在剩下的范围继续随机抽取一个地址，就可以保证所指向的数据依旧不会重复。
